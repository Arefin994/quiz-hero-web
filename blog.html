Q-1: What is the difference between map, filter, find?

map: It transforms each element by applying a function to it then returns a new unit with the transformed values.
filter: This selects elements based on a condition by applying a function to each element of returns a new one with the elements that meet the condition(s).
find: returns the index of the first element in array/object that satisfies a/some condition(s).

Q-2: What is the difference between local Storage, section Storage and Cookies?

Local Storage: stores key-values in the browser's storage that stays even after the browser is closed.
Session Storage: stores key-values in the browser's storage that are cleared when the browser is closed.
Cookies: stores small data on the client-side that can be accessed by both the client-side and server-side. You can set an expiration date also.

Q-3: What is event loop? How does it work?

The event loop allows the runtime environment tob handle multiple tasks in an asynchronous way. It continuously check the call stack for tasks to execute. If there are no tasks in the call stack, it checks the task queue for new tasks. If there are tasks in the queue, it pushes them onto the call stack to be executed.

Q-4: What quizTimer function does?

This function takes a parameter and and if the parameter is true then it runs clearInterval function. If false it runs setInterval function. Q-5: What chooseQuiz function does?

The function first checks if the question exists. If it exists, then search the index of the answer. Then updates the object at that index with the new answer using splice method. If the it doesn't exists then it add new object to answer array. Then it calls displayAnswer function to display the answers that was updated.
